# Archive

These files preserve development history, retired code, test results, and setup approaches that are no longer part of the event workflow.

**Do not use archive files as workshop instructions. They may contain outdated paths, hardware mappings, tag IDs, and commands.**

Current event documents begin at the repository [README](../../README.md).

## What Is Archived

- Root development notes, status reports, and architecture references
- Unused root configuration, packaging, and installation helpers under `root_legacy/`
- Superseded setup and troubleshooting documents
- Deprecated robot entry points
- Older demo, navigation, hardware test, and retired skill code under `code/`

The archive is retained for maintainers who need historical context. Participants do not need anything in this folder.
